If you see small ^(-) or `v(+) ' symbols on the edge of the menu, this means that there are more things left unshowed above/under the current item due to insufficient screen space. In the text display box, the number of text above the current position is shown as a percentage in the bottom right corner. Use ** Up / Down  ** Arrow** to scroll by line,** PageUp/ PageDown** to scroll the entire page.

Select **OK** in the menu to confirm the operation it controls. Select **Cancel** to cancel the operation, and then usually return to the top one menu. Use **TAB** to move the cursor and select the button.

Most interfaces have ** Help** buttons -- please use them! It usually provides useful context-related tips, and if you're not sure what to do in a configuration menu, choose **Help**!

**Special function**

== sync, corrected by elderman ==

If the initial letter of the menu item name is the only one, you can select the menu item by entering it. Typically, it is the number of the item.

The console driver contains a roll-back buffer to view the contents of a rolling screen. To use rollback, press **Scroll Lock** on the keyboard, and then use the arrow key or **Page Up/Page Down** to view the saved text. To quit rollback mode, press **Scroll Lock** again. This feature is very useful for viewing start-up information (temporarily try!) and also applies to the processing of subs that do not use menus and that often scroll to the top of the screen, Shell, or other " expert mode " .

FreeBSD also supports multiple "virtual consoles" that you can use for multiple event sessions at the same time. Toggle screens using **ALT + F < Numbers**, where `F < Numbers' is a functional key to the screen you want to see. By default, the system has 8 virtual consoles: you can edit `/etc/ ttys ' files to add the relevant vty entry `" `off' field changed to `on' to enable more virtual consoles (up to 12).
