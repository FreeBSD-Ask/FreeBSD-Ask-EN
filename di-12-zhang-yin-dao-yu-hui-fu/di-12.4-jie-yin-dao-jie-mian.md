When coming out of the FreeBSD menu, press **ESC**, out of the guide. There will be a hint __CODESPAN_0_. Enter __CODESPAN_1 __ to see all supported resolution lists:


```sh
OK gop list
mode 0: 1920x1080x32, stride=1920
mode 1: 640x480x32, stride=640
mode 2: 800x600x32, stride=800
mode 3: 1024x768x32, stride=1024
mode 4: 1280x720x32, stride=1280
mode 5: 1280x1024x32, stride=1280
```

HERE, I CHOOSE CODESPAN_0 TO MEASURE THE EFFECTS:

```sh
OK gop set 0
```

Right away。

If appropriate, continue to direct:

```sh
OK menu
```

```sh
exec="gop set 0"
```


## __CODESPAN_0 (UEFI) OR __CODESPAN_1 (BIOS)

According to the document [LOADER.CONF(5)] (https://man.freebsd.org/cgi/man.cgi?loader.conf(5)), two variables accept the following values:

```sh
值	           分辨率
480p	        640x480
720p	        1280x720
1080p	       1920x1080
1440p	       2560x1440
2160p	       3840x2160
4k	          3840x2160
5k	          5120x2880
宽x高        宽x高
```

__CODESPAN_0_:

Write __CODESPAN_0_ into __CODESPAN_1_, the effect after restart is consistent with the gop method。
